# Signal Declaration Documentation Index

This directory organizes the project into readable layers for research, benchmarks, proof notes, and app/UI development.

## Canonical order

1. `theory/` — conceptual and mathematical framework.
2. `benchmarks/` — deterministic test interpretation and metric notes.
3. `whitepapers/` — long-form publishable drafts.
4. `app/` — UI/dashboard planning notes.

The goal is to keep the repository usable from SmartIDE, GitHub, Termux/proot, and future desktop workflows without depending on REPL state or temporary shell history.
